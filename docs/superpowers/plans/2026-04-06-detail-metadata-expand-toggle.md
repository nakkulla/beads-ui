# Detail Metadata Expand Toggle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> superpowers:subagent-driven-development (recommended) or
> superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let users click a metadata path in the detail sidebar to expand it
inline for full reading, then click again to collapse it.

**Architecture:** Keep the change entirely inside the detail view. Add small
local expanded-state tracking keyed by metadata row label, render the metadata
value as a toggle control with `aria-expanded`, and split CSS into collapsed vs
expanded styles. Cover the interaction with focused detail view tests.

**Tech Stack:** lit-html, vitest, vanilla JS DOM API, existing detail view CSS

---

### Task 2: Implement inline expand/collapse

**Files:**

- Modify: `app/views/detail.js`
- Modify: `app/styles.css`
- Verify with: `app/views/detail.test.js`

- [ ] **Step 1: Add local expanded-state tracking in detail view**

Track expanded metadata rows with a local `Set<string>` keyed by `Spec` / `Plan`
/ `Handoff`, and clear it whenever a new issue is loaded.

- [ ] **Step 2: Render metadata value as a toggle control**

Make each metadata value clickable, add `aria-expanded`, and apply a modifier
class when expanded.

- [ ] **Step 3: Add CSS for collapsed vs expanded states**

Keep the current ellipsis style by default, and switch to wrapped multiline
display when expanded.

- [ ] **Step 4: Run focused tests**

Run: `npx vitest run app/views/detail.test.js` Expected: PASS.
